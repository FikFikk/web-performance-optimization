Halaman Demo - Sebelum Optimasi Script Pihak Ketiga

[Gambar LCP / Hero Image Halaman]

Halaman ini memuat script pihak ketiga secara synchronous dan tidak optimal. Script analitik dimuat langsung di <head>, yang memblokir parser HTML browser dan menunda Render Pertama (FCP).

Selanjutnya, sebuah script Widget Chat Pelanggan berukuran besar dan memiliki performa buruk dimuat di bagian bawah body. Ini memblokir utas utama (Main Thread) selama lebih dari 350ms, menyebabkan Total Blocking Time (TBT) tinggi dan responsivitas interaksi buruk (meningkatkan risiko keterlambatan metrik Interaction to Next Paint (INP)).

Masalah yang Terjadi:

  • Parser Blocking: Script analytic tanpa defer/async menghentikan DOM parsing.
  • CPU Blocking: Script chat mengeksekusi inisialisasi yang berat di awal page load meskipun user belum tentu menggunakannya.
  • Network Congestion: Kompetisi memperebutkan bandwidth jaringan antara hero HTML/CSS dan script chat.
